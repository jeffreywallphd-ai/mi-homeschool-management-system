## Metadata

Substantive canonical docs must begin with:

- Status.
- Last reviewed.
- Canonical for.
- Related ADRs.
- Related docs.
- Related tests.
- Supersedes.

## Status Values

- proposed: useful guidance, not final authority.
- accepted: current authority.
- superseded: replaced by a later document or ADR.
- deprecated: retained for history, not current guidance.

## Update Rule

When behavior changes, update the relevant canonical doc and any affected context pack in the same change.

## Link Rule

Docs should link to the most relevant canonical sources rather than duplicating large sections of text.
