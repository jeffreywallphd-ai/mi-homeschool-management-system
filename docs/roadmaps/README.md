## Purpose

This directory contains implementation roadmaps for bounded slices of work.

Product direction and long-term sequencing belong in `docs/product/roadmap.md`. Implementation roadmaps belong here when they translate accepted product, domain, architecture, and standards decisions into phased build plans.

## Rules

- Roadmaps are planning documents, not permission to bypass accepted ADRs or contracts.
- Each roadmap should name scope, non-scope, phases, exit criteria, and verification gates.
- If a roadmap needs a deferred decision, it must call that out instead of quietly deciding it.
