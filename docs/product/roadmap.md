# Roadmap

- Status: proposed
- Last reviewed: 2026-06-06
- Canonical for: high-level implementation sequencing
- Related ADRs: [ADR-0003](../adr/ADR-0003-records-and-credentials-module-from-start.md)
- Related docs: [V1 Scope](v1-scope.md), [Non-Goals](non-goals.md)
- Related tests: not yet implemented
- Supersedes: none

## Phase 0: Documentation and Architectural Foundation

- Establish docs governance.
- Establish product/domain/legal/architecture standards.
- Record initial ADRs.
- Define AI-agent context routing.

## Phase 1: Core Local Record Model

- Household, school profile, student, school years, terms.
- Requirement sets and Michigan seed.
- Courses, curriculum plans, lessons.
- Assignments, submissions, stored files.

## Phase 2: Evidence, Grades, Credits, and Graduation

- Gradebook.
- Rubrics, tests, progress evaluations.
- Credit policies and awards.
- Graduation plan and progress.

## Phase 3: Records and Documents

- Report cards.
- Transcripts.
- Diplomas.
- Course-description packets.
- Portfolio exports.
- Graduation packets.

## Phase 4: Backup, Restore, and Archive Hardening

- Manual backup.
- Restore validation.
- Student archive export.
- Automatic backup scheduling.

## Phase 5: Refinement

- Better document templates.
- Student-facing workflows.
- Additional jurisdiction profiles.
- Optional import/export conveniences.

## Roadmap Constraint

The roadmap is not permission to implement future phases out of order when doing so would weaken contracts or bypass accepted decisions.
